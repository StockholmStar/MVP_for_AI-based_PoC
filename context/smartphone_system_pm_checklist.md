# Smartphone System PM Checklist

- Define system entry points: Launcher, Lock screen, Notification shade, Settings, Home screen, or -1 screen.
- Clarify owner boundaries across System UI, Settings, Launcher, app teams, platform services, data/logging, and QA.
- Document permission, privacy, redaction, and user control behavior.
- Cover loading, empty, error, no permission, disabled, offline, and success states.
- Consider performance, battery, memory, stability, cold start, and low-memory behavior.
- State region, device tier, Android version, OTA, and OEM skin constraints.
- Include privacy-safe analytics events only when they are necessary for product decisions.
