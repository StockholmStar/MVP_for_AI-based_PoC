# Sample PRD Shape

## Product Overview
Briefly explain the user problem, product value, and system surface.

## Goals and Non-goals
Separate measurable v1 goals from future ideas.

## Functional Requirements
Use stable requirement IDs and map each one to UX/prototype coverage.

## Key States and Edge Cases
Document loading, empty, error, no permission, disabled, offline, and success states.

## Acceptance Criteria
Write criteria that QA and engineering can verify.
