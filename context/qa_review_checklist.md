# QA Review Checklist

- Validate normal path, empty state, loading state, error state, disabled state, no permission state, and success feedback.
- Verify cross-owner integration points and persistence across reboot or OTA when relevant.
- Check regression risk for latency, battery, memory, notification correctness, and privacy.
- Confirm acceptance criteria are measurable and mapped to PRD requirements.
- Include negative tests for revoked permissions, stale data, storage failure, and unsupported devices.
